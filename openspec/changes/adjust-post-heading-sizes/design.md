## Context

PaperMod テーマのデフォルト CSS では `.post-title`（記事タイトル）と `.post-content h1`（本文中の `#` 見出し）がともに `font-size: 40px` に設定されており、視覚的な区別がない。執筆者が Markdown のセマンティクス通りに `#` から見出しを書くと、タイトルと同じサイズの文字が本文中に現れてしまう。

変更対象は `assets/css/extended/custom.css`（プロジェクト固有の CSS 上書きファイル）のみ。テーマファイル（`themes/papermod/`）は変更しない。

## Goals / Non-Goals

**Goals:**
- 本文中の h1〜h3 フォントサイズを記事タイトル（40px）より小さく設定する
- `#` から見出しを書き始める自然な執筆体験を実現する
- テーマファイルを変更せず、メンテナンスコストを最小化する

**Non-Goals:**
- h4〜h6 のサイズ変更（使用しない運用のため対象外）
- レスポンシブ対応の追加（現状のテーマに zmedia.css での見出し上書きがないため不要）
- フォントウェイト・行間・マージンの調整

## Decisions

### CSS 上書きを `custom.css` に追記する

**決定**: テーマファイルを変更せず、`assets/css/extended/custom.css` に追記する。

**理由**: テーマファイルを変更すると `themes/papermod/` のアップデート時に差分が失われる。`custom.css` はプロジェクト固有の上書きとして Hugo がマージするため、テーマ更新の影響を受けない。

### サイズスケールの選択

**決定**: 以下のサイズに設定する。

| セレクタ | 現在 | 変更後 |
|----------|------|--------|
| `.post-title`（記事タイトル） | 40px | 変更なし |
| `.post-content h1` | 40px | 32px |
| `.post-content h2` | 32px | 24px |
| `.post-content h3` | 24px | 20px |

**理由**: 本文テキストが 18px であることを踏まえ、各レベルで明確な差を持たせる。タイトルから h1 への落差（▼8px）、h1 から h2（▼8px）、h2 から h3（▼4px）という段階的なスケール。h3 は 20px で本文（18px）より 2px 大きいが、太字とマージンで視覚的な区別は十分確保できる。

**検討した代替案**: h1 のみを調整し h2/h3 は現状維持する案も検討したが、h1（32px）と h2（32px）が同じサイズになってしまうため、h2 以下も連動して調整する。

## Risks / Trade-offs

- [リスク] h3（20px）と本文（18px）の差が 2px と小さい → h3 は太字＋マージン（`margin: 24px 0 16px`）があるため、実際の可読性への影響は軽微
- [リスク] 既存記事で `#` を使っていると見た目が変わる → 意図した変更であり、h1 が小さくなることで本文との区別がより明確になる（改善方向）
