## Context

記事一覧ページは PaperMod テーマのカード型レイアウトを使用している。各記事は `background: var(--entry)` + `border` + `border-radius` でカードとして描画され、ページ背景 `.list` は `var(--code-bg)` でカードを浮かせている。ユーザー作成記事は `custom.css` で `.post-self` クラスに青い背景色を付与して区別している。

この変更ではカード装飾を廃止してフラットなリストにし、ユーザー記事の強調を左ボーダーに切り替える。

## Goals / Non-Goals

**Goals:**
- 記事一覧をフラットなリストとして表示する（カード背景・枠線・角丸の廃止）
- ユーザー作成記事を左ボーダーアクセントで視覚的に区別する
- ライトモード・ダークモード両対応

**Non-Goals:**
- テーマの fork や大規模カスタマイズ（最小限の CSS 変更で実現する）
- 記事単体ページ（single.html）のスタイル変更
- first-entry（トップ記事の特別表示）のスタイル変更

## Decisions

### D1: カード背景の廃止方法

`post-entry.css` の `.post-entry` から `background`, `border`, `border-radius` を削除する。テーマファイルを直接編集する。

代替案: `custom.css` でオーバーライドする → 元のスタイルが残り続けるため、テーマ側を直接変更する方が明確。このプロジェクトではテーマを submodule ではなく直接管理しているため問題ない。

### D2: ページ背景色の統一

`theme-vars.css` の `.list` / `.dark.list` で設定されている背景色（`--code-bg` / `--theme`）を削除する。カードを浮かせるための背景色差が不要になるため。

### D3: 左ボーダーの色

ライトモード: `rgba(60, 130, 210, 0.45)` — グレースケール基調のデザインに対して柔らかく目立つ青
ダークモード: `rgba(100, 170, 245, 0.50)` — 暗い背景で視認性を確保するため若干明るく

代替案: `var(--primary)` や `var(--secondary)` を使う → グレースケールに溶け込みすぎてアクセントとして弱い。テーマの無彩色パレットとの対比で有彩色（青系）を使うことで、少量でも視認性を確保する。

### D4: 記事間の区切り

カード枠の代わりに `margin-bottom` の余白で区切る。区切り線（`border-bottom`）は入れない。

## Risks / Trade-offs

- [テーマ直接編集] テーマアップデート時にコンフリクトの可能性 → このプロジェクトでは PaperMod を直接管理しており、upstream 追従はしていないため実質的なリスクは低い
- [視認性] 左ボーダーだけで十分に区別がつくか → ユーザー記事は 137 件中 1 件と少数であり、ボーダーは十分に目立つ。今後記事が増えても青のアクセントは識別しやすい
