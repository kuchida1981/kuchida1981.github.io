- [ ] 1.1. `.github/workflows/daily-post.yaml` を修正します。
- [ ] 1.2. `create-pull-request` アクションに `labels` パラメータを追加し、`AI-generated` および `automerge-24h` を含めます。

## 2. 自動マージワークフローの作成
- [ ] 2.1. `.github/workflows/automerge.yaml` に新しいファイルを作成します。
- [ ] 2.2. 1時間ごとのスケジュール（`cron: '0 * * * *'`）で実行される新しいワークフローを定義します。
- [ ] 2.3. `gh` CLI を使用して `automerge-24h` ラベルを持つPRを検索するロジックを実装します。
- [ ] 2.4. 各PRの作成時刻をチェックし、24時間以上経過していればマージするステップを追加します。全てのステータスチェックが合格している場合にのみPRがマージされるようにします。

## 3. ドキュメンテーション
- [ ] 3.1. 必要に応じて、プロジェクトの `README.md` またはその他の関連ドキュメントを更新し、新しい自動化プロセスを反映させます。(自己修正: この範囲では、別途ドキュメントを更新する必要はないかもしれませんが、検討する価値はあります)。

## 4. 検証
- [ ] 4.1. `daily-post.yaml` ワークフローを手動でトリガーし、正しいラベルが付与されたPRが作成されることを確認します。
