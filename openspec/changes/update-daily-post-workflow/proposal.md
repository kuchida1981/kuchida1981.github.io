# 変更: 自動マージによる日次投稿ワークフローの更新

## 目的
AIが生成したブログ記事のレビューおよびマージプロセスを効率化するため、以下の2点の改善を導入します。
1. プルリクエストのラベルによってAI生成コンテンツであることを明確にします。
2. 24時間のレビュー期間後、これらのPRを自動的にマージすることで、手動での介入を削減します。

## 変更内容
- **プルリクエストのラベリング:**
    - `.github/workflows/daily-post.yaml` ワークフローが変更されます。
    - `create-pull-request` アクションは、新しく作成される各PRに `AI-generated` および `automerge-24h` の2つのラベルを付与するようになります。

- **自動マージ:**
    - 新しいGitHub Actionsワークフローファイル `.github/workflows/automerge.yaml` が作成されます。
    - このワークフローは、スケジュールに基づいて（例: 1時間ごと）実行されます。
    - `automerge-24h` ラベルを持つオープンなプルリクエストを検索します。
    - 見つかった各PRについて、作成から24時間以上が経過しているかを確認します。
    - レビュー期間が経過し、かつ全てのステータスチェックが成功している場合、ワークフローはプルリクエストを自動的にマージします。

## 影響範囲
- **変更されるファイル:**
    - `.github/workflows/daily-post.yaml`
- **新規ファイル:**
    - `.github/workflows/automerge.yaml`
- **影響を受ける仕様:**
    - 自動化されたリポジトリワークフローをカバーする新しい機能 `github-actions` が定義されます。
